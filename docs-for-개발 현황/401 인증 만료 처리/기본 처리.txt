# 401 인증 만료 처리

## 문제
- 앱 시작 시 저장된 토큰으로 /api/auth/me 호출
- 토큰 만료 시 401 에러 발생
- 콘솔에 에러 로그 출력됨

## 해결 방법
- 401 에러를 예상된 상황으로 처리
- 토큰 삭제 후 로그인 페이지로 리다이렉트

## 수정 파일
- src/app/_layout.tsx

## 처리 흐름
1. 앱 시작 시 SecureStore에서 accessToken 확인
2. 토큰 있으면 authApi.getMe() 호출
3. 401 에러 발생 시:
   - accessToken, refreshToken 삭제
   - 사용자 상태 초기화 (clearMe)
   - 로그인 페이지로 이동 (router.replace)
4. 다른 에러는 콘솔에 로그 출력

## 핵심 코드
```typescript
} catch (error) {
  const axiosError = error as AxiosError;
  if (axiosError.response?.status === 401) {
    console.log("[RootLayout] Token expired, redirecting to login...");
    await SecureStore.deleteItemAsync("accessToken");
    await SecureStore.deleteItemAsync("refreshToken");
  } else {
    console.error("[RootLayout] Failed to initialize user:", error);
  }
  clearMe();
  router.replace("/(auth)/login");
}
```
