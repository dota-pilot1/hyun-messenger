# 기본 페이지 추가 및 라우터 테스트 요약

## 1. 새로운 페이지 파일 생성
- 경로: `app/test1.tsx`
- 특징: `app/` 폴더 바로 밑에 생성하면 주소는 자동으로 `/test1`이 됩니다.

## 2. 페이지 UI 구현
- `ThemedView`, `ThemedText` 등을 사용하여 기본적인 UI를 구성했습니다.
- `Stack.Screen` 컴포넌트를 사용하여 페이지 상단 타이틀을 '테스트 페이지'로 설정했습니다.
- `useRouter` 훅의 `router.back()`을 사용하여 뒤로 가기 기능을 추가했습니다.

## 3. 라우터 테스트 (이동 링크 추가)
- 경로: `app/(tabs)/index.tsx` (홈 화면)
- 내용: `Link` 컴포넌트를 사용하여 `/test1`으로 이동하는 버튼을 추가했습니다.
  ```tsx
  <Link href="/test1">
    <ThemedText type="link">테스트 페이지로 이동하기</ThemedText>
  </Link>
  ```

## 4. 결과 확인
- 홈 화면 하단에 추가된 'Step 4' 섹션의 링크를 누르면 `test1.tsx` 화면으로 이동합니다.
- 파일 시스템 기반 라우팅이 정상적으로 작동함을 확인했습니다.
